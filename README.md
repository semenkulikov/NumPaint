## NumPaint

Преобразование произвольных изображений в аккуратные раскраски по номерам
с контролируемым количеством цветов, чёткими контурами и легендой.

### Возможности

- **Квантование цветов в Lab**: палитра подбирается в перцептивном пространстве, поэтому
  исходная цветопередача сохраняется лучше, чем при простом RGB-k-means.
- **Сегментация по цветам**: для каждого цвета строятся маски и сглаженные контуры,
  мелкие артефакты можно отсечь через `--min-region`.
- **Контуры и номера**: генерируется контурное изображение с толстыми чёрными линиями
  и номерами внутри областей.
- **Легенда с палитрой**: сбоку рисуется легенда: номер → цвет → RGB/hex.
- **Высокое разрешение**: сегментация делается на уменьшенной копии, но результат
  дорисовывается на полном размере исходника.
- **CLI и Python-API**: можно использовать как утилиту командной строки или как библиотеку.

### Установка

```bash
python -m venv .venv
.venv\Scripts\activate  # Windows
pip install -r requirements.txt
```

Требования: Python 3.10+.

### Использование: CLI

Базовый пример (Windows, из корня проекта):

```bash
set PYTHONPATH=src
python -m numpaint path/to/image.jpg --colors 24 --max-size 1024
```

- **`--colors`**: сколько уникальных цветов оставить в палитре (кластеры k-means).
- **`--auto-colors`**: подобрать количество цветов автоматически по ошибке в Lab.
- **`--min-colors` / `--max-colors` / `--target-mse`**: диапазон и целевая ошибка для авто-подбора.
- **`--max-size`**: максимальная сторона изображения для внутренней обработки (для скорости).
- **`--min-region`**: минимальная площадь области (в пикселях); по умолчанию подбирается автоматически от размера кадра.
- **`--out-dir`**: базовый каталог; по умолчанию используется `result/<basename>`.
- **`--svg`**: сохранить векторный контур (`*_outline.svg`) с номерами.
- **`--pdf`**: сохранить контур в PDF (`*_outline.pdf`).
- **`--no-preview`** / **`--no-outline`**: отключить сохранение соответствующих файлов.
- **`--log-level`**: уровень логов (`DEBUG`, `INFO`, ...).

Что получается в `result/<basename>`:

- `image_outline.png` — контур с номерами и легендой сбоку.
- `image_preview.png` — постеризованное цветное превью.
- `image_palette.json` — палитра: номер → BGR/RGB/hex.
- (опц.) `image_outline.svg` — векторный контур с номерами.
- (опц.) `image_outline.pdf` — PDF-версия контура.

### Использование: Python-API

```python
import cv2
from numpaint import Config, generate_paint_by_numbers

cfg = Config(colors=24, max_size=1024, min_region_area=80)
result = generate_paint_by_numbers("path/to/image.jpg", cfg)

if result.outline_image is not None:
    cv2.imwrite("outline.png", result.outline_image)

if result.preview_image is not None:
    cv2.imwrite("preview.png", result.preview_image)

print(result.palette_hex)  # список hex-цветов вида ['#RRGGBB', ...]
```

### Telegram-бот

1. Создай файл `.env` в корне проекта:

```env
BOT_TOKEN=твой_токен_бота
LOG_LEVEL=INFO
RESULT_DIR=result
NUMPAINT_MAX_SIZE=1024
NUMPAINT_DEFAULT_COLORS=24
```

2. Запусти бота (из корня проекта, `venv` уже активирован):

```bash
python -m bot.main
```

Команды в боте:

- `/start` — приветствие и краткая инструкция.
- `/echo` — эхо всех текстовых сообщений.
- `/download` — запускает сценарий: пришли фото/изображение → бот вернёт `outline.png` с раскраской.

Результаты обработки для бота складываются в подпапки вида `result/user_<id>_<timestamp>/`.

### Алгоритм (кратко)

1. **Нормализация**: загружаем изображение, приводим к BGR, уменьшаем до `max_size`
   по большей стороне для внутренней обработки.
2. **Квантование**: переводим в Lab и запускаем MiniBatchKMeans на подвыборке пикселей;
   строим карту меток и палитру центров кластеров.
3. **Сегментация**: по каждой метке строим бинарную маску, прогоняем через морфологию,
   находим контуры, фильтруем слишком мелкие области, при желании упрощаем контуры.
4. **Рендер**: рисуем белый холст, обводим контуры чёрным, считаем центр области и
   ставим номер цвета, генерируем легенду (цветной прямоугольник + номер + hex).
